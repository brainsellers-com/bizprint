# sync-rules プレースホルダマーカー

`.claude/rules/` 内のルールファイルには `<!-- {{XXX}} -->` 〜 `<!-- END {{XXX}} -->` 形式のマーカーが含まれる場合がある。
これらは sync-rules スキルがPJ固有コンテンツを保護するための境界マーカーである。

## MUST NOT

- `<!-- {{XXX}} -->` マーカー行を削除してはいけない
- `<!-- END {{XXX}} -->` マーカー行を削除してはいけない
- マーカーの書式（`<!-- {{` と `}} -->`）を変更してはいけない
- マーカーを「不要なHTMLコメント」として除去してはいけない

## なぜマーカーが必要か

sync-rules がルールテンプレートを更新する際、マーカーで囲まれた範囲のコンテンツ（PJが記入した固有情報）を保持し、範囲外のルール本文のみを上書きする。マーカーが消えると、次回の sync-rules 更新時にPJ固有コンテンツがテンプレートのデフォルトで上書きされる。

## マーカー内のコンテンツの扱い

- `⚠️ sync-rules placeholder` で始まる行は未記入を示すデフォルトテキスト。PJ固有の値に置き換えた場合、このデフォルトテキストは削除してよい（マーカー行自体は残すこと）
- `<!-- 例: ... -->` は記入例のHTMLコメント。記入後に削除してよい（マーカー行自体は残すこと）
- PJ固有の値を記入済みの場合、その内容を編集・追記するのは問題ない（マーカー行自体は残すこと）
